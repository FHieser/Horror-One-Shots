## Structure

Each one-shot follows this structure:

```
one-shot-name/
├── README.md           # Overview, themes, player count, duration
├── introduction.md     # Setting, premise, opening narration
├── mechanics.md        # Special rules, entities, horror mechanics
├── story-flow.md       # Plot structure, acts, pacing
├── *.csv               # Tabular data (roles, rooms, NPCs, items)
└── assets/             # Images, maps, handouts
```

## CSV Conventions

- Use UTF-8 encoding
- First row is always headers
- Use semicolons (;) for lists within cells
- Keep descriptions concise, use markdown files for extended details
